Dataset analyzed: parkings.csv (sourced from EMT Madrid Open Data). This document summarizes how the dataset satisfies the six Linked Data requirements (R1–R6).

Requirement-by-Requirement Evaluation

Requirement Interpretation Assessment Evidence / Notes
R1. Smart city domain Dataset should belong to a smart-city-related domain (e.g., mobility, parking, and utilities). Satisfied This dataset is published in the EMT open-data catalog under the mobility/parking group ("Aparcamientos EMT").
R2. Available as CSV Data must be available in CSV format. Satisfied A CSV resource (parkings.csv) exists for this dataset and the uploaded file is a CSV. The file used for this analysis is the CSV copy.
R3. Open license Data must have an open license permitting publishing/reuse. Satisfied The dataset metadata indicates a Creative Commons Attribution license (CC‑BY), which allows sharing and reuse with attribution.
R4. Linkable to real-world entities Data should include fields that allow linking to locations or other generic entities. Satisfied The CSV contains explicit geographic coordinates (lat, long) and address/town fields, so records can be linked to places and other spatial datasets (reverse geocoding, spatial joins, OSM, municipal street registers, etc.).
R5. Documentation exists (optional) Metadata, README or PDF describing the dataset is available. Satisfied (available metadata) The dataset has metadata and a PDF of metadata; metadata entries include last update timestamps and resource list (CSV, GeoJSON, KML, SHP, etc.).
R6. Data can come from multiple sources (optional) Optional: data may be assembled/linked from multiple sources. Partially satisfied The dataset itself appears to be a single EMT-managed resource (one authoritative source). However, it can readily be combined with other sources (municipal street registers, traffic counts, parking occupancy time-series) to produce multi-source datasets.

Preliminary Resource Naming Strategy

The dataset will adopt a persistent URI pattern following Linked Data best practices. Ontology terms will be identified using the namespace https://data.smartcitymadrid.es/ontology/parking#, while individual resources will be identified under https://data.smartcitymadrid.es/resource/parking/{id}. This approach ensures consistency, persistence, and dereferenceability of resources across the project and full alignment with the final Resource Naming Strategy defined in the analysis document.

Practical Notes for Reuse

  • Attribution: Because of CC‑BY 4.0, reuse must include attribution to EMT / EMT data portal.
  • Spatial joins: Use lat,long to join to GIS layers (GeoJSON/KML resources are also available in the same dataset for convenience).
  • Time dimension: This CSV is a static snapshot listing aparcamiento attributes. If you need historical availability or occupancy you should use the time-series parking availability datasets (published separately by EMT).
  • Combining sources (R6): To create multi-source datasets, combine these records with municipal registries, traffic sensors, or parking transaction logs — ensure license compatibility and preserve attribution.
