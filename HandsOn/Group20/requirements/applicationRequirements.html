Parking Finder Application — Requirements and UI Mockups

1. Application Overview

The application is developed using the Aparcamientos EMT dataset (EMT Madrid Open Data), analyzed in the Data Source Analysis and Dataset Requirements documents. It enables users to explore parking facilities across Madrid, offering search and filtering options and displaying results on an interactive map with detailed information for each facility.

2. Functional Requirements

  • Search by name: Users can enter text to find parking lots by matching it against the facility name.
  • Search by location: Users can enter a street address to locate nearby parking facilities, or click on the map to retrieve the closest ones to that point.
  • Filter options: Users can filter parking lots based on whether they are managed by EMT.
  • Map view: Display parking facilities on an interactive map with markers positioned according to their coordinates (lat, long).
  • List view: Display parking lots in a scrollable list showing their name and distance.
  • Detail view: Each parking facility displays its main attributes, including its identifier, family, name, email, street address, postal code, autonomous community, category, category code, URL icon, coordinates, city, EMT status, type, and Wikidata links.
  • Favorites: Users can save parking lots as favorites for quick access and remove them at any time.
  • Attribution: The app must show: “Data sourced from EMT Madrid Open Data (Aparcamientos EMT dataset), licensed under CC-BY 4.0”.

3. Non-Functional Requirements

  • Performance: Fast search and filtering, with results generally displayed in under two seconds.
  • Usability: Clear and intuitive interface in which both the side panel and the map provide interactive search and result exploration features.
  • Scalability: Ability to accommodate additional layers in the future (such as traffic data or occupancy sensors).
  • Internationalization: Current support for Spanish, with the possibility of extending to additional languages.

4. User Interface Description

The user interface is structured into three sections:

  1. Side panel: Search bar, EMT filter, nearby-results list, and access to favorites.
  2. Map area: Interactive map that displays parking facilities with markers and allows users to initiate searches by clicking on it.
  3. Detail view: Presents detailed information about the selected parking lot.

6. Interaction Flow

  1. The user enters a name or a street address and presses "Buscar".
  2. The map displays the matching results and the side panel updates accordingly.
  3. The user clicks a marker or list item to view its details.
  4. The user may add/remove the parking facility from favorites.
  5. If the user clicks on the map, the closest parking lots are retrieved and listed.

7. Consistency with Dataset and Ontology

This application has been designed in full alignment with the dataset (parkings.csv), the ontology defined at https://data.smartcitymadrid.es/ontology/parking#, and the resource naming strategy described in the Data Source Analysis document. All entities displayed in the interface correspond to Linked Data resources under https://data.smartcitymadrid.es/resource/parking/{id}.
