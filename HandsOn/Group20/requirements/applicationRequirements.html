Parking Finder Application — Requirements and UI Mockups

1. Application Overview

The application is developed using the Aparcamientos EMT dataset (EMT Madrid Open Data), analyzed in the Data Source Analysis and Dataset Requirements documents. It enables users to explore parking facilities across Madrid, filter by parking type, and view results on a map or in a list, with detailed information for each facility.

2. Functional Requirements

  • Search by type: Users can select parking lot categories (rotation, residents, park-and-ride, etc.).
  • Search by location: Users can enter an address or use geolocation to find nearby parking lots.
  • Filter options: Users can filter parking lots based on their characteristics (e.g., residents-only, rotation).
  • Map view: Display parking lots on a map with markers based on coordinates (lat,long).
  • List view: Display parking lots in a scrollable list with name, address, type, and distance.
  • Detail view: Each parking lot shows address, type, and a link to EMT page.
  • Favorites: Users can save favorite parking lots for quick access.
  • Attribution: The app must show: “Data sourced from EMT Madrid Open Data (Aparcamientos EMT dataset), licensed under CC-BY 4.0.”

3. Non-Functional Requirements

  • Performance: Fast search and filtering (under 2s for results).
  • Usability: Clear interface, mobile-friendly layout, accessible color contrast.
  • Scalability: Ability to add additional datasets (traffic, occupancy sensors) in the future.
  • Internationalization: Support for Spanish (default) and English.

4. User Interface Description

The user interface will have three main sections:

  1. Home/Search Screen: Search bar, type filter dropdown, location input, and quick access buttons.
  2. Results Screen: Map view with pins for parking lots and a toggle for list view.
  3. Detail Screen: Displays details of the selected parking lot with options to save as favorite.

6. Consistency with Dataset and Ontology

This application has been designed in full alignment with the dataset (parkings.csv), the ontology defined at https://data.smartcitymadrid.es/ontology/parking#, and the resource naming strategy described in the Data Source Analysis document. All entities displayed in the interface correspond to Linked Data resources under https://data.smartcitymadrid.es/resource/parking/{id}.
