Aparcamientos EMT — Data Source Analysis, Licensing & Resource Naming Strategy

Dataset analyzed: parkings.csv (sourced from EMT Madrid Open Data). This document summarizes a detailed analysis of the data source, its licensing aspects, and the Resource Naming Strategy defined for the dataset to be generated.

Data Source Summary
  • Source: EMT Madrid Open Data — Aparcamientos EMT (https://datos.emtmadrid.es/dataset/aparcamientos-emt)
  • Format: CSV, GeoJSON, KML, SHP
  • Geolocation: Latitude/longitude provided for each record
  • Domain: Smart Mobility / Parking
  • License: Creative Commons Attribution 4.0 (CC-BY 4.0)

1. Data Source Analysis

The Aparcamientos EMT dataset contains information about public and private parking facilities in Madrid managed by EMT. Each record provides attributes such as the facility name, address, coordinates, category, and type. The dataset was analyzed to assess its structure, quality, and potential for integration.

1.1 Quantitative Overview

The following quantitative summary highlights the dataset's structure and key statistical characteristics:

  • Rows: 90
  • Columns: 17 (id, family, name, address, areaCode, zrea, category, catCode, country, email, URLIcono, long, lat, state, town, emtParking, type)
  • Coordinates: Complete (100% of rows contain valid lat and long values)
  • Missing values: areaCode ≈ 59%, email ≈ 82%
  • Spatial coverage: All records located within the city of Madrid (lat 40.36 to 40.49; long –3.78 to –3.60)

1.3 Relationships and Data Quality

  • Each parking lot (Parking) belongs to one city (Madrid).
  • Administrative districts can be inferred from coordinates.
  • Addresses are in free-text format; normalization is recommended for future data integration.
  • All coordinate pairs are valid; ready for use in map-based applications.
  • Potential additional fields (capacity, hours) could be added later to enrich the model.

2. Licensing Analysis

The dataset is distributed under the Creative Commons Attribution 4.0 International (CC-BY 4.0) license. This license allows reuse, modification, and redistribution provided that attribution is given to the original source (EMT Madrid).

  • Allowed: Share, adapt, and combine this data for any purpose, including commercial use.
  • Requirements: Attribution to EMT Madrid and indication of modifications if any.
  • Recommended Attribution:
    “Data sourced from EMT Madrid Open Data (Aparcamientos EMT dataset), licensed under CC-BY 4.0.”

Proposed License for Derived Dataset: The derived dataset, which may integrate EMT data with other open resources, should also adopt the CC-BY 4.0 license to preserve openness and compatibility. Therefore, the dataset and its RDF representations remain fully open, reusable, and interoperable.

3. Resource Naming Strategy

The Resource Naming Strategy (RNS) defines a consistent and persistent way to identify all ontology terms and individual resources within the dataset. It ensures full alignment between the ontology, the generated RDF data, and potential future linked datasets derived from this work.

This strategy follows the official Linked Data best practices, using hash URIs for ontology terms (classes and properties) and slash URIs for individual instances. This approach guarantees clarity, persistence, and dereferenceability of resources on the Web.

The naming pattern has been designed in accordance with the “Guidelines for the Generation of Linked Data — Ontology and Preparation” (OEG-UPM, 2014), ensuring that URIs are stable, human-readable, and compliant with Linked Data principles.

Resource Type Naming Pattern Example Notes
Ontology terms (classes, properties) https://data.smartcitymadrid.es/ontology/parking# https://data.smartcitymadrid.es/ontology/parking#Parking For classes and properties in the ontology (hash URI).
Individuals (instances) https://data.smartcitymadrid.es/resource/parking/{localName} https://data.smartcitymadrid.es/resource/parking/arguelles For individual parking facilities (slash URI).
