1) Ontology Overview

This ontology models the Barcelona Metro transportation system, capturing metro lines, stations, their relationships, operational status, and accessibility information. It supports the interactive metro map application by providing a semantic layer for querying and visualizing metro network data.

Namespace https://data.example.org/transport/bcn/metro/ontology#

Prefix metro:

Resource Base https://data.example.org/transport/bcn/metro/resource/

Design Principles

  • Reusability: Classes and properties follow W3C best practices for linked data
  • Extensibility: Schema supports future additions (e.g., bus lines, tram networks)
  • Interoperability: Uses standard vocabularies (GeoSPARQL, Dublin Core) where applicable
  • Clarity: Property names use camelCase for readability; classes use PascalCase

2) Core Classes

The ontology defines 8 main classes representing entities in the metro system:

MetroLine

Class

Description: Represents a metro line (e.g., L1, L2) with visual attributes and route information.

  • lineID xsd:integer - Unique identifier
  • lineCode xsd:integer - Display code
  • lineName xsd:string - Line name (e.g., "L1")
  • lineDescription xsd:string - Route description
  • originStation xsd:string - Starting station
  • destinationStation xsd:string - End station
  • lineColor xsd:string - Primary hex color
  • auxiliaryColor xsd:string - Secondary hex color
  • textColor xsd:string - Text hex color
  • lineOrder xsd:integer - Display order
metroRes:metroLine/L1

Station

Class

Description: Represents a metro station with location and operational data.

  • stationID xsd:integer - Unique identifier
  • stationCode xsd:integer - Station code
  • stationName xsd:string - Station name
  • inauguratedDate xsd:date - Opening date
  • latitude xsd:decimal - Geographic latitude
  • longitude xsd:decimal - Geographic longitude
  • hasGeometry geo:wktLiteral - WKT point
metroRes:station/sagradaFamilia

StationLine

Class

Description: Represents the relationship between a station and a line (many-to-many).

  • stationOrder xsd:integer - Position on line
  • pictogram xsd:string - Visual identifier
  • lineColor xsd:string - Associated color
metroRes:stationLine/sagradaFamilia_L2

TransportOperator

Class

Description: Organization operating metro lines (e.g., TMB, FGC).

  • rdfs:label xsd:string - Operator name
  • rdfs:comment xsd:string - Full name/description
metroRes:operator/tmb

TransportFamily

Class

Description: Type of transport service (Metro, Bus, Tram, etc.).

  • rdfs:label xsd:string - Family name
  • familyOrder xsd:integer - Display order
metroRes:family/metro

AccessibilityType

Class

Description: Accessibility level of a station (Accessible, Not Accessible).

  • rdfs:label xsd:string - Accessibility status
metroRes:accessibility/accessible

StationStatus

Class

Description: Operational status (Operational, Under Construction, Closed).

  • rdfs:label xsd:string - Status description
metroRes:status/operational

ServiceType

Class

Description: Type of service provided on a station-line relationship.

  • serviceOrigin xsd:string - Service origin
  • serviceDestination xsd:string - Service destination
metroRes:serviceType/regular
