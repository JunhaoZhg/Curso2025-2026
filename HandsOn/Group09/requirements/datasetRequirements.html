Dataset Requirements Analysis

This document analyzes whether the selected datasets for the Museums of Madrid Explorer application satisfy the defined requirements (R1–R6).

Mandatory Requirements (R1–R4)

Requirement Description Dataset Satisfaction
R1 Data are in one of the selected smart city domains. [x] The selected datasets belong to the culture and tourism domain, specifically museum information within the city of Madrid.
R2 Data are available as a CSV file. [x] The datasets are provided in CSV format, ensuring they can be easily processed and visualized.
R3 Data have an open license so they can be published. [x] The datasets are released under open licenses from the Madrid Open Data portal, allowing reuse and publication.
R4 Data can be easily linked with generic real-world entities (e.g., locations). [x] The datasets include addresses and geographic coordinates, enabling clear linkage to real museum locations across Madrid.

Optional Requirements (R5–R6)

Requirement Description Degree of Satisfaction
R5 Documentation exists for the data. Partial: The museum dataset includes basic metadata, but some fields lack detailed descriptions, requiring extra verification.
R6 Data can come from multiple data sources. Partial: The main dataset comes from the municipal open data portal, though additional cultural datasets (e.g., exhibitions, visitor statistics) could be integrated for richer insights.

Conclusion

The selected datasets fully satisfy the mandatory requirements (R1–R4) for the Museums of Madrid Explorer application. The optional requirements (R5–R6) are partially met, and future additions from other cultural datasets could strengthen completeness and contextual detail.
